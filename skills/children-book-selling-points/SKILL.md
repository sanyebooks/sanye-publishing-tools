# 图书卖点拆解（童书版）

把童书拆成有原文依据、能打动家长、能直接用于渠道执行的营销卖点表。默认交付 `.xlsx`，不要只给泛泛的童书内容分析。

## 开始前

1. 读取 `references/input-and-evidence.md`，建立童书事实卡和证据池。
2. 读取 `references/children-title-formulas.md`，根据适读年龄、家长痛点和书内卖点选择标题公式。
3. 读取 `references/output-standard.md`，遵守字段、圈层、星级、标题和 Excel 标准。
4. 生成 Excel 后运行 `scripts/validate_titles.py <xlsx路径>`，超长标题或双表不一致时不得交付。

## 默认输出

默认生成30条卖点；用户指定数量时服从用户要求。工作簿包括：

1. `卖点表`
   - `选题/标题`
   - `网感化后的标题`
   - `书里的具体内容`
   - `星级`
   - `具体页码`
   - `针对的人群圈层`
   - `人群圈层说明`
2. `小红书笔记&图像提示词`（用户要求内容种草时生成）
   - `序号`
   - `网感化后的标题`
   - `小红书笔记文案`
   - `封面图AI创作提示词`
   - `配图1 AI创作提示词`
   - `配图2 AI创作提示词`
   - `配图3 AI创作提示词`
3. `人群圈层说明`

## 工作流程

### 1. 建立童书事实卡

记录书名、作者、童书类型、内容主题、篇幅结构、阅读方式、已验证的适读年龄、用户确认的重点投放年龄、阅读阶段、文字与拼音情况、互动工艺、材质特性、已知家长痛点和页码体系。区分出版方明示信息、用户确认信息、书内证据和营销画像。材料只给宽泛适读范围时，可在范围内为不同卖点选择与场景最匹配的单点年龄作为营销画像，但不得把该单点年龄写成出版方适读结论或真实用户案例。

### 2. 建立童书证据池

优先提取：具体故事、角色选择、知识点、阅读任务、游戏或互动机制、语言难度、拼音与字号、图文比例、反复阅读价值、亲子共读场景和可验证的产品工艺。每条卖点保留页码或章节位置。

### 3. 确定购买者、使用者与圈层

标题主要写给家长或其他购买决策者，卖点证据落到孩子的真实阅读体验。只使用 `核心层`、`扩展层`、`潜力层`，默认至少55%的卖点服务核心层。

### 4. 生成并筛选卖点

每条卖点同时包含：一个具体的家长决策问题、本书独有内容、孩子可能获得的阅读价值与证据位置。合并同义项，不得把“提升认知”“培养习惯”换词凑数。

### 5. 生成童书网感标题

将 `卖点表` B 列作为整个工作簿标题的唯一数据源。先提炼家长购买理由，再根据卖点选择一种主公式；不得把故事情节换一种说法就当作网感标题。优先保留最强的2—3个元素，不要把整条公式硬塞进标题。常规目标长度为12—18个可见字符，硬上限为20个可见字符。

标题至少命中以下三项中的两项，且必须包含第2项或第3项：

1. 精准人群钩子：`X岁女儿`、`X岁儿子`、`X岁娃`、`幼儿园孩子`、`一年级孩子`等。
2. 家长能立刻识别的具体问题或购买场景。
3. 本书独有解法或孩子可观察到的阅读价值。

年龄或阶段有证据时，让50%—70%的标题以前置人群钩子锁定目标家庭，优先使用自然口语化的 `X岁女孩／X岁男孩／X岁女儿／X岁儿子／X岁娃`，其余标题保持公式多样性。性别与卖点无关时在女孩、男孩、女儿、儿子、孩子之间轮换，不得暗示某种能力或问题只属于某一性别。

禁止在标题中使用 `3—8岁`、`2-6岁`、`6至9岁` 等宽泛年龄范围；它们保留在适读信息和正文中。标题应选择范围内一个与卖点最匹配的单点年龄，并补充具体身份与心理、行为或家庭场景，例如 `7岁男孩嫌弃自己内向没朋友`。单点年龄是营销画像，不代表缩窄本书的官方适读范围。

单点年龄必须落在已验证适读范围内，并与卖点的成长阶段合理匹配。无法确认任何适读范围时，可改用有证据的阅读阶段；年龄和阶段都无法确认时不得编造。没有真实家庭使用案例时，不得虚构“我女儿看了X遍”“陪儿子读了半年”等第一人称经历；可使用目标画像式表达，如 `7岁男孩嫌弃自己内向没朋友`。

逐条执行“购买理由测试”：只读标题，家长是否能看出它在解决谁的什么问题，或者为什么这本书值得买？如果只能看出人物做了什么、故事发生了什么，判定为内容概括并重写。

### 6. 评定营销优先级

`星级`只允许为空、`加星`、`超级加星`。优先标注家长痛点强、适读对象清晰、购买理由具体且书内支撑充分的卖点。

### 7. 生成渠道内容

用户需要时才生成小红书笔记和4张图提示词。Sheet3 的 `网感化后的标题` 按 `序号` 原样复用对应主表 B 列标题，不得另行改写。封面图优先呈现精准年龄或阅读阶段、一个核心利益点、书籍或亲子阅读实景以及清晰留白。

### 8. 验证交付物

检查：

- 每条卖点是否能指回书内具体内容。
- 适读年龄、阅读效果、销量、奖项、推荐人和孩子反应是否有证据。
- 页码是否真实；无法确认时是否写成 `待核页码：章节/位置线索`。
- 主表 B 列是否全部不超过20个可见字符，且没有机械截断。
- 标题是否通过购买理由测试；纯人物动作、情节复述或主题概括是否已重写。
- B列是否完全没有宽泛年龄范围表达。
- 年龄证据充足时，是否有50%—70%的标题以前置的单点年龄与孩子身份快速锁定目标家庭。
- Sheet3 标题是否与对应主表标题逐字一致。
- 公式是否服务于书内事实，而不是用爆款话术掩盖证据不足。
- Excel 是否冻结表头、开启筛选、自动换行、列宽可读并完成渲染检查。

## 不得这样做

- 不得编造适读年龄、孩子的阅读反应、家长反馈、时间周期、能力变化、销量、奖项或权威背书。
- 不得使用“保证提分”“一定戒掉手机”“几天变天才”等无证据或绝对化承诺。
- 不得把“认知起飞”“长脑子”“省妈神器”当作任何童书都能套用的空洞标签。
- 不得机械复制报告中的爆款标题或为30条卖点套用同一句式。
- 不得为主表和 Sheet3 生成两套不同标题。
