# 图书卖点拆解

把一本书拆成有原文依据、能够传播、能够指导渠道执行的营销卖点表。默认生成可交付的 `.xlsx`，不要只给一段泛泛分析。

## 开始前

1. 读取 `references/input-and-evidence.md`，确认材料是否足以支撑卖点与页码。
2. 读取 `references/output-standard.md`，遵守字段、圈层、星级和文案标准。
3. 当用户要求 Excel 时，使用可用的电子表格能力创建、检查并渲染工作簿。
4. 不因材料不足停止全部工作；标明证据强弱和待核位置，并说明需要补充什么。

## 默认输出

默认生成30条卖点；用户指定数量时服从用户要求。工作簿包括：

1. `卖点表`
   - `选题/标题`
   - `网感化后的标题`
   - `书里的具体内容`
   - `星级`
   - `具体页码`
   - `针对的人群圈层`
   - `人群圈层说明`
2. `小红书笔记&图像提示词`（用户需要内容种草时生成）
   - `序号`
   - `网感化后的标题`
   - `小红书笔记文案`
   - `封面图AI创作提示词`
   - `配图1 AI创作提示词`
   - `配图2 AI创作提示词`
   - `配图3 AI创作提示词`
3. `人群圈层说明`
   - `圈层`
   - `定义`
   - `典型人群`
   - `使用建议`

## 工作流程

### 1. 建立图书事实卡

先提取并记录：书名、作者、品类、核心主题、章节结构、出版阶段、已知目标读者、可用页码体系和材料缺口。区分用户提供的事实与推断。

### 2. 建立证据池

从书中寻找能够成为购买理由的具体内容：观点、方法、案例、数据、冲突、场景、金句含义、人物经历、清单和反常识结论。为每条证据保留页码或章节位置。

如果是小说，优先提取“世界观规则＋具体情节后果”，不要把主题判断当成书内事实。每条卖点至少落到一个人物、场景、制度或命运转折；标题可以制造悬念，但证据栏要说明实际发生了什么。

### 3. 划分人群圈层

只使用 `核心层`、`扩展层`、`潜力层`。先说明每个圈层为什么会购买或传播，再将卖点匹配到圈层。默认至少55%的卖点服务核心层，除非材料明显支持更广市场。

### 4. 生成并筛选卖点

每条卖点必须同时包含：具体读者问题、书中独有内容、可传播表达和证据位置。合并同义项，避免为了凑数量重复改写。先生成候选池，再筛选最终结果。

### 5. 评定营销优先级

`星级`只允许为空、`加星`、`超级加星`。不要把所有卖点都标星；优先选择强痛点、强购买理由、强传播表达且书内支撑充分的卖点。

### 6. 生成渠道内容

仅在用户需要时生成小红书文案和4张图提示词。文案应从卖点和原文证据生长出来，不得批量套用同一个开头、结构或结尾。

### 6.1 小说剧透控制

小说交付物增加剧透口径：

- `预热期`优先使用世界观、人物处境与前半程冲突，不揭示关键死亡和最终结局。
- `上市期`可披露局部转折，但单条物料不要串起完整命运链。
- `深度传播期`才使用结局、终极命题与人物闭环，并明确标注“含剧透”。
- 卖点表可保留高价值剧透项供编辑选择，但应单列使用说明，不能默认全部对外发布。

### 7. 验证交付物

检查：

- 每条卖点是否能指回书中具体内容。
- 页码是否真实；无法确认时是否标成 `待核页码：章节/位置线索`。
- 是否存在换词不换意的重复项。
- 标题是否夸大或超出书稿证据。
- 核心层比例和星级分布是否合理。
- Excel 是否冻结表头、自动换行、列宽可读并设置数据验证。
- 是否已经渲染并目视检查主表和至少一个内容页。
- 小说是否区分预热、上市与深度传播阶段的剧透边界。

## 不得这样做

- 不得编造原文、数据、作者经历或精确页码。
- 不得输出适用于任何书的空泛卖点，如“治愈自己”“提升认知”，除非写清本书独有的实现路径和证据。
- 不得把章节标题简单改写成营销卖点。
- 不得把营销判断伪装成已验证的市场事实。
- 不得为了标题冲击力歪曲原意。
